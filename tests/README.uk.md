# 🧪 Система Автоматизованого Тестування (`tests/`)

Ця папка містить набір автоматизованих тестів, написаних за допомогою фреймворку `pytest`. Вони гарантують стабільність критичних компонентів системи та запобігають регресіям під час розробки нових фіч.

## 🎯 Що ми тестуємо?

### 1. `test_database.py` (Інтеграційні тести)
Перевірка Data Access Layer. Гарантує надійне підключення до PostgreSQL, перевіряє CRUD-операції та роботу транзакцій (включно з відкотом у разі помилки).

### 2. `test_core_analytics.py` (Юніт-тести)
Валідація математичної правильності алгоритмів фільтрації, агрегації та кластеризації (K-Means) для пошуку аномалій.

### 3. `test_ml_model.py` (Валідація ML Pipeline)
Не тренує моделі (щоб зекономити час), але перевіряє, чи коректно завантажуються ваги (`.h5` / `.onnx`) і скейлери, та чи видає інференс моделі правильні формати тензорів.

### 4. `test_physics.py` (Тести предметної області)
Суворі юніт-тести для фізичного рушія Digital Twin. Валідує термодинамічні формули, розрахунки втрат потужності та залежність опору від температури.

### 5. `test_security.py` (Аудит безпеки)
Перевірка системи на стійкість до вразливостей, гарантуючи, що всі SQL-запити параметризовані (захист від SQL Injection), а дані від користувача проходять очистку.

---

## 🚀 Як запустити тести

Для запуску всього набору тестів переконайтеся, що віртуальне середовище активоване, та виконайте команду з кореня проєкту:

```bash
pytest tests/ -v
```

Для тестування конкретного модуля:
```bash
pytest tests/test_physics.py -v
```
