# 📱 Інструкція Користувача (User Manual)

Ласкаво просимо до інтерфейсу **Energy Monitor ULTIMATE**. Цей гід допоможе вам ефективно керувати дашбордом та інтерпретувати дані.

---

## 🛠️ Сайдбар: Керування та Фільтри

Сайдбар — це ваш центр управління. Він розділений на три логічні зони:

### 1. Вибір джерела даних
*   **Синтетична модель (Smart City)**: Режим у реальному часі (Digital Twin). Дані генеруються симулятором.
*   **Еталонні дані (Kaggle)**: Режим дослідження історичних даних великих енергомереж.

### 2. Глобальні фільтри
*   **Регіон**: Масштабування всієї аналітики до конкретного регіону (наприклад, Київ).
*   **Діапазон дат**: Вибір часового вікна для історичного аналізу.
*   **Підстанція**: Фокусування на конкретному об'єкті для ML-прогнозу.

### 3. Керування симуляцією
*   **Кнопка "▶️ Запустити Симуляцію"**: Активує фоновий процес Digital Twin на 15 хвилин.
*   **Чекбокс "Жива карта"**: Вмикає auto-refresh для гео-візуалізації.

---

## 🗂️ Навігація сторінками (8 Вкладок)

Дашборд розділений на спеціалізовані вкладки для різних ролей (від оператора до фінансового директора).

### 1. 🗺️ Карта мережі
Візуалізація енергомережі на карті.
*   **Точки**: Підстанції. Розмір залежить від навантаження.
*   **Лінії**: Магістралі. Колір вказує на відсоток завантаженості (Зелений → Жовтий → Червоний).

### 2. 📉 Споживання
Основний аналітичний блок споживання.
*   **Групування**: Можливість змінювати агрегацію (По регіонах / По підстанціях).
*   **Графіки**: Динаміка MW у часі.

### 3. 🏭 Генерація
Огляд джерел енергії.
*   **Структура**: Розподіл (Атомна/Теплова/Відновлювана) у вигляді кругових діаграм.
*   **Баланс**: Співвідношення виробленої та спожитої енергії.

### 4. 🚨 Журнал аварій
Оперативне управління інцидентами.
*   **Smart Editor**: Можливість змінювати статус аварії прямо в таблиці (Pending → Processing → Resolved).
*   **Пріоритезація**: Автоматичне сортування за критичністю (Critical/Warning).

### 5. 💰 Економіка
Фінансові показники мережі.
*   **Виторг**: Розрахунок потенційного доходу на базі поточних тарифів.
*   **Втрати**: Фінансова оцінка енерговтрат у лініях.

### 6. 🤖 AI Аналітика
Глибокий аналіз даних за допомогою ШІ.
*   **Кластеризація**: Групування об'єктів за поведінковими характеристиками (K-Means).
*   **Anomaly detection**: Виявлення нетипової активності.

### 7. 🔮 Прогноз ШІ (Найголовніше)
LSTM-прогнозування на наступні 24 години.
*   **Графік**: Поєднує реальну історію (синя лінія) та ШІ-прогноз (оранжева пунктирна лінія).
*   **Довірчий інтервал**: Напівпрозоре поле, що вказує на ймовірну похибку моделі.

### 8. 📜 Цифровий архів (OLAP)
Доступ до історичних зрізів даних.
*   **Експорт**: Можливість завантажити відфільтровані дані у CSV/Excel для стороннього аналізу.

## 💡 Поради щодо інтерпретації

*   **Health Score < 20%**: Критичний стан трансформатора. Потребує негайної перевірки у вкладці "Аварії".
*   **H2 PPM > 500**: Ознака перегріву або деградації масла.
*   **Gaps у прогнозі**: Якщо оранжева лінія занадто відхиляється від синьої, зверніть увагу на **Mean Absolute Error (MAE)** у підписі графіка.

## 🆘 Troubleshooting для користувача

*   **Немає даних на карті?** Переконайтеся, що вибрано "Синтетична модель" та запущено симуляцію.
*   **Помилка прогнозу?** Перевірте, чи вибрана конкретна підстанція (прогноз не працює для режиму "Усі").
*   **Інтерфейс "тупить"?** Спробуйте натиснути `R` для повного перезавантаження кешу Streamlit.
