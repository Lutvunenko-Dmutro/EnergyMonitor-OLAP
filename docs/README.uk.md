# 📚 Технічна документація та Архітектура (`docs/`)

Ця папка містить всю технічну та архітектурну документацію проєкту. Проєкт використовує строгий підхід "Documentation as Code", гарантуючи, що документація розвивається синхронно з кодом.

## 🗺️ Система ProperDocs (Паспорти системи)
Ми використовуємо власну систему "паспортів" модулів під назвою **ProperDocs** (розміщено в `system/map/`).
Кожен важливий Python-файл має відповідний Markdown-паспорт з детальним описом його призначення.

**Що включає паспорт:**
- Визначення компонента та його зона відповідальності.
- Архітектура потоків даних (Data flow).
- Інтерфейси та залежності.
- Відомі обмеження або шляхи для майбутнього оновлення.

Ці паспорти можна автоматично перевірити на цілісність, запустивши скрипт `scripts/audit_passports.py` через Developer Dashboard.

## 🎨 UI/UX Макети
Папка `ui/` містить згенеровані HTML-версії інтерфейсу та структурні огляди фронтенду. Вони використовуються для узгодження дизайну перед його імплементацією в Streamlit.

- `ui.html`: Головний макет додатку.
- `ui/views/`: Макети окремих екранів (Finance, Generation, Alerts).

## 🚀 Підтримка актуальності
Щоб гарантувати, що документація завжди актуальна, розробники повинні запускати скрипт `rebuild_master_index.py` після додавання нових паспортів. Цей скрипт автоматично оновить головне дерево навігації.
