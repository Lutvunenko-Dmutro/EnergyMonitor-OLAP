# 🧠 ML Map: ШІ-Конвеєр (LSTM Pipeline)

Цей розділ описує «інтелект» системи — процес від обробки сирих даних до формування високоточного прогнозу на 24 години.

---

## 📄 predict_v2.py
**Роль:** Глобальний контролер ШІ-прогнозування.

Це оркестратор, який з'єднує дані, моделі та алгоритми виправлення помилок.

### Ключові технології:
1.  **ONNX Inference:** Використання формату ONNX для швидкого виконання моделей незалежно від фреймворку навчання.
2.  **Domain Adaptation (Масштабування):** Алгоритм `_compute_scale_factor` автоматично підлаштовує глобальну модель під конкретну потужність вибраної підстанції.
3.  **Bias Correction (Зшивання):** Технологія `_apply_bias_correction_and_blend` забезпечує безшовний перехід між реальними даними та початком прогнозу, усуваючи "стрибки" на графіку.

---

## 📄 vectorizer.py
**Роль:** Математична підготовка тензорів (Feature Engineering).

Перетворює часові ряди у формат, який «розуміє» нейронна мережа.

### Особливості обробки:
- **Periodic Time Signals:** Перетворення годин та днів тижня у тригонометричні функції ($\sin$ та $\cos$). Це дозволяє моделі розуміти, що 23:00 та 00:00 знаходяться поруч.
- **Sliding Window:** Формування вхідного вікна на 48 годин для аналізу ретроспективи перед прогнозом.
- **Multimodal Input:** Вектор включає не тільки навантаження, а й температуру масла, вміст газів (H2) та погодні умови.

---

## 📄 metrics_engine.py
**Роль:** Аудитор точності та статистичний двигун.

Відповідає за те, щоб ми могли довіряти прогнозам ШІ.

### Метрики та тести:
- **RMSE / MAE / MAPE / R²:** Стандартні наукові метрики для оцінки відхилення прогнозу від факту.
- **Statistical Audit:** Використання тесту **Shapiro-Wilk** для перевірки нормальності розподілу помилок ( residuals). Якщо помилки не нормальні — модель потребує перенавчання.
- **Outlier Masking:** Алгоритм 3-сигма для фільтрації сенсорного шуму під час бектестування.

---

## 📄 model_loader.py
**Роль:** Менеджер ресурсів ШІ.

Забезпечує «ліниве» завантаження ваг моделей та об'єктів Scaler (нормалізаторів), щоб не перевантажувати пам'ять сервера.
