# 🏭 Services Map: Digital Twin та Симуляція

Цей розділ описує фонові сервіси проєкту, які забезпечують роботу системи в реальному часі, імітуючи поведінку фізичних об’єктів енергосистеми.

---

## 📄 sensors_db.py
**Роль:** Фоновий колектор телеметрії (Digital Twin).

Це і є той самий «двигун», який оживляє дашборд під час презентації.

### Життєвий цикл (Lifecycle):
1.  **Singleton Guard:** Використовує файл `logs/sensors.lock`, щоб запобігти запуску кількох копій симулятора одночасно.
2.  **Telemetry Loop:** Кожні 5 секунд генерує нові покази для всіх підстанцій (навантаження, частота, температура масла, водень).
3.  **State Broadcaster:** Записує актуальний стан у `logs/live_state.json`, звідки інтерфейс Streamlit миттєво зчитує дані для візуалізації.
4.  **Auto-Shutdown:** Має вбудований таймер (зазвичай 15 хв), після якого сервіс автоматично вимикається для економії ресурсів.

---

## 📄 data_generator.py
**Роль:** Генератор синтетичного навантаження та початкових даних.

Відповідає за те, щоб база даних PostgreSQL не була порожньою і містила реалістичні історичні тренди.

### Алгоритми генерації:
- **Baseline Generation:** Створює історію навантаження на основі професійних профілів споживання (Residential, Industrial, Commercial).
- **Physical Consistency:** Використовує функції з `physics.py` для того, щоб температура трансформаторів та вміст водню логічно залежали від згенерованого навантаження.
- **Random Anomaly Injector:** З певною ймовірністю додає в дані «стрибки» або «просідання», щоб перевірити систему сповіщень (Alerts).

---

## 📄 db_seeder.py
**Роль:** Ініціалізація структури даних.

Використовується при першому розгортанні проєкту для наповнення таблиць `Regions`, `Substations` та `Generators` базовими параметрами.
