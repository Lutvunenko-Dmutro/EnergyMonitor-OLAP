# 📂 Паспорт Системи Кешування: `data/fallback/`

> **Шлях до директорії:** `data/fallback/`

## 📌 Призначення директорії
Директорія `data/fallback` є серцем підсистеми стійкості до відмов **NeonGuard Resilience Layer**. Вона призначена для зберігання локальних копій результатів SQL-запитів в оптимізованому двійковому форматі `Parquet`.

## ⚙️ Принцип роботи (Offline Mode)
1. **Аналіз запитів:** При виконанні SQL-запиту через `run_query()`, система генерує MD5-хеш запиту.
2. **Гаряче кешування:** Результати успішних запитів записуються у файл `query_[MD5_HASH].parquet`.
3. **Автономне відновлення:** У разі виникнення помилок з'єднання з хмарною базою даних (наприклад, Neon Cloud "холодний старт" або відсутність інтернету), система автоматично перемикається в офлайн-режим і завантажує дані з локального Parquet-файлу.

## 📊 Характеристики активів кешу
* **Формат збереження:** Apache Parquet (колоночне бінарне стиснення).
* **Переваги:** Зменшення споживання оперативної пам'яті (RAM) до 80% порівняно з CSV, збереження строгої типізації колонок, миттєве завантаження.
* **Автоочищення:** Керується підсистемою гігієни оперативної пам'яті та кешу системи.
