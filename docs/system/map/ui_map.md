# 🎨 UI Map: Користувацький Інтерфейс (Streamlit)

Цей розділ описує «обличчя» системи — те, як організовано взаємодію з користувачем та візуалізацію даних у реальному часі.

---

## 📄 main.py
**Роль:** Точка входу та оркестратор сесій.

Цей файл ініціалізує додаток, налаштовує конфігурацію сторінки та керує бічною панеллю (sidebar).

### Ключові функції:
- **setup_page:** Встановлює назву вкладки браузера, іконку та режим макету (wide).
- **Session State Management:** Керує глобальними станами — вибраним регіоном, підстанцією та набором даних (Live vs Kaggle).

---

## 📄 ui/segments/dashboard.py
**Роль:** Головний архітектор інтерфейсу.

Тут зосереджена логіка динамічного перемикання між вкладками.

### Технологія Streamlit Fragments:
Проєкт використовує декоратори `@st.fragment`, що дозволяє:
- Оновлювати «Живу карту» або блок KPI незалежно від решти сторінки.
- Економити оперативну пам'ять (фрагменти не копіюють весь DataFrame при кожному оновленні).
- Забезпечити плавну анімацію «дихання» інтерфейсу (run_every=5s).

---

## 📄 ui/views/
**Роль:** Тематичні модулі представлення даних.

Кожен файл у цій папці — це окрема бізнес-функція системи:

1.  **kpi.py:** Блок телеметрії в реальному часі (навантаження, частота, стабільність).
2.  **forecast.py:** Складний інтерфейс ШІ-прогнозування з можливістю симуляції погоди.
3.  **map.py:** ГІС-візуалізація вузлів мережі на базі Folium.
4.  **historical_audit.py:** Інтерфейс для OLAP-аналізу історичного архіву з фільтрацією за будь-яким виміром.
5.  **alerts.py:** Інтерактивний журнал аварій з можливістю редагування статусів.

---

## 📄 ui/components/styles.py
**Роль:** Глобальна естетика проєкту.

Містить кастомні CSS-стилі, які створюють «Dark Mode» та ефект скляного морфізму (glassmorphism) для карток метрик.
