# ⚙️ Мапа Ядра (Core)

Цей розділ описує «фундамент» системи — від того, як зберігаються дані, до математичних моделей фізики енергомереж.

## 📂 База Даних (`src/core/database/`)

### 📄 [loader.py](https://github.com/Lutvunenko-Dmutro/EnergyMonitor-OLAP/blob/main/src/core/database/loader.py)
**Навіщо:** Головний шлюз для зчитування даних у Streamlit.
- **Як працює:** Використовує `st.cache_data` для миттєвого завантаження великих обсягів (зберігає DataFrame у пам'яті).
- **Чому так:** Забезпечує відсутність затримок при перемиканні між вкладками.

---

## 📂 Аналітика та Фільтрація (`src/core/analytics/`)

### 📄 [filter.py](https://github.com/Lutvunenko-Dmutro/EnergyMonitor-OLAP/blob/main/src/core/analytics/filter.py)
**Навіщо:** Центр керування тим, що користувач бачить на екрані.
- **Як працює:** Виконує векторну фільтрацію за часом, регіонами та підстанціями.
- **Чому так:** Дозволяє працювати з мільйонами рядків через Pandas без перевантаження процесора.

---

## 📂 Фізична Модель (`src/core/physics.py`)

**Навіщо:** Це «цифрове серце» енергосистеми. Тут живуть формули, які перетворюють сухі цифри на реальну фізику.

### 🧮 Ключові алгоритми:
1. **Розрахунок втрат (`calculate_line_losses`):**
   - **AC лінії:** Втрати ростуть квадратично ($I^2 \cdot R$).
   - **HVDC лінії:** Втрати лінійні, що робить їх ідеальними для магістралей.
2. **Оцінка стабільності (`estimate_grid_stability`):**
   - Розраховує баланс генерації/споживання. Якщо споживання > 120% від генерації — статус "Критично".
3. **Симуляція погоди (`calculate_weather`):**
   - Використовує синусоїду для добового циклу температури та нормальний розподіл (jitter) для інерції.
