# ⚙️ Core Map: Аналітичне Ядро системи

Цей розділ присвячений внутрішній аналітичній логіці проєкту. Тут відбуваються фізичні розрахунки, агрегація даних та взаємодія з базою даних.

---

## 📄 physics.py
**Роль:** Фізико-математичний двигун системи.

Цей файл відповідає за симуляцію реальних процесів в енергосистемі.

### Ключові алгоритми:
1.  **Розрахунок втрат (calculate_line_losses):**
    - Для **AC** ліній (змінний струм) втрати розраховуються за квадратичною залежністю: $Losses \propto I^2 \times R$.
    - Для **HVDC** ліній (постійний струм високої напруги) — за більш лінійною моделлю, що відображає їх вищу ефективність на великих відстанях.
2.  **Діагностика трансформаторів (calculate_transformer_health):**
    - Розраховує здоров'я на основі навантаження, температури масла та концентрації водню (H2) у ppm.
3.  **Фінансова модель (calculate_energy_price):**
    - Реалізує динамічне ціноутворення згідно з постановами НКРЕКП, враховуючи нічні та пікові тарифи.

---

## 📄 aggregator.py
**Роль:** Підготовка та нормалізація даних для аналітики та ШІ.

Файл забезпечує консистентність даних перед тим, як вони потраплять на графіки або в нейромережу.

### Ключові функції:
- **aggregate_consumption:** Перетворює сирі виміри у погодинні зрізи методом `resample('1h').mean()`.
- **add_relative_load:** Обчислює навантаження у відсотках (%) відносно історичного максимуму підстанції. Це критично для навчання LSTM моделей на різних за потужністю об'єктах.

---

## 📄 queries.py
**Роль:** Бібліотека SQL-запитів для OLAP-системи.

Тут зберігаються складні аналітичні запити до PostgreSQL, які формують багатовимірні зрізи даних.

### Основні запити:
- **QUERY_LOAD_WEATHER:** Об'єднує (JOIN) дані про навантаження підстанцій з погодними звітами за координатами та часом.
- **QUERY_FINANCE:** Розраховує вартість переданої електроенергії в реальному часі на основі тарифів та фактичного навантаження.
- **QUERY_ALERTS:** Вибирає журнал аварійних подій з географічною прив'язкою до регіонів.

---

## 📄 loader.py
**Роль:** Шлюз доступу до даних.

Реалізує функцію `get_verified_data`, яка є єдиним джерелом істини для всього UI. Вона забезпечує кешування та верифікацію цілісності даних.
