# РОЗДІЛ 4. ОПИС ПРОЄКТНИХ РІШЕНЬ ТА ПРОГРАМНОЇ РЕАЛІЗАЦІЇ

## 4.1. Архітектура інтелектуальної системи EnergyMonitor-OLAP

У ході виконання індивідуального завдання було спроєктовано 4-рівневу архітектуру системи, що базується на принципах модульності та масштабованості:

1.  **Рівень даних (OLAP Layer):** Реалізований на базі хмарної PostgreSQL (Neon). Використовує схему «зірка» (Star Schema) для швидкої агрегації телеметрії підстанцій.
2.  **Інтелектуальне ядро (AI Layer):** Складається з моделей LSTM v3, які виконують прогноз навантаження на основі 24-годинного «вікна» історичних даних.
3.  **Фізичний рушій (Digital Twin):** Модуль `physics.py`, що розраховує теплову деградацію обладнання та втрати в лініях за законом Джоуля-Ленца.
4.  **Користувацький інтерфейс (UI Layer):** Веб-дашборд на базі Streamlit для візуалізації KPI, прогнозів та геопросторових даних.

## 4.2. Опис бази даних та інформаційного забезпечення

Інформаційне забезпечення системи базується на реляційній моделі даних. Основні сутності включають:
*   **Substations:** Реєстр енергетичних об'єктів та їх характеристик.
*   **LoadMeasurements:** Таблиця фактів, що містить мільйони записів телеметрії.
*   **WeatherReports:** Дані про температуру та погодні умови для врахування кореляції навантаження.

## 4.3. Реалізація предиктивної моделі LSTM

Для реалізації прогнозування було розроблено нейронну мережу з такою структурою:
*   **LSTM шар (128 юнітів):** виокремлення часових ознак.
*   **Dense шари з ReLU активацією:** нелінійна трансформація.
*   **Тригонометричне кодування часу:** використання sin/cos функцій для годин та днів тижня, що забезпечує безперервність циклічних процесів.

За результатами тестування на еталонному наборі даних PJM Interconnection досягнуто похибку прогнозу **MAPE в діапазоні 1.5% – 3.1%**, що підтверджує ефективність обраних рішень.
