# 4 ВИСНОВКИ

Під час проходження переддипломної практики на кафедрі «Інформаційних та комунікаційних технологій некомп'ютерних наук та інженерії програмного забезпечення» ЗВО «МНТУ» (20.04.2026 – 16.05.2026) мною було проведено комплексне дослідження за тематикою кваліфікаційної роботи «Прогнозування часових рядів енергоспоживання для вдосконалення технологій Smart City на основі рекурентних нейронних мереж».

**Основні результати виконання індивідуального завдання практики:**

1. **Опрацьовано теоретичну базу.** Вивчено концепції Smart City, Smart Grid та Digital Twin відповідно до міжнародних стандартів ISO/ASME 23247:2021, IEEE 1547-2018 та IEEE C57.91. Обґрунтовано доцільність застосування рекурентних нейронних мереж архітектури LSTM для предиктивного аналізу мультисезонних нелінійних часових рядів енергоспоживання.

2. **Проаналізовано існуючі рішення.** Проведено порівняльний аналіз класів систем SCADA, комерційних аналітичних платформ (AutoGrid, C3.ai) та відкритих рішень (InfluxDB+Grafana, OpenEMS). Встановлено, що жодне з розглянутих рішень не поєднує реалтайм-моніторинг підстанцій, ML-прогнозування, фізичне моделювання деградації обладнання та доступний SaaS-інтерфейс в єдиній платформі. Це обґрунтовує оригінальність та практичну цінність розроблюваної системи.

3. **Проведено порівняльний аналіз методів прогнозування.** Зіставлено точність моделей ARIMA (MAPE=8.3%), XGBoost (MAPE=5.1%) та LSTM (MAPE=3.1%) на еталонному датасеті PJM Interconnection. На основі аналізу обрано архітектуру LSTM як оптимальну для задачі прогнозування навантаження на горизонт до 48 годин.

4. **Спроєктовано чотирирівневу архітектуру** системи EnergyMonitor-OLAP: шар даних (PostgreSQL Neon, схема «зірка»), інтелектуальне ядро (LSTM v3, 9 ознак, вікно 48 кроків), фізичний рушій (Digital Twin підстанції: температура масла, H₂, Health Score) та рівень представлення (Streamlit-дашборд).

5. **Верифіковано результати.** На тестових наборах даних підтверджено працездатність обраних алгоритмів. Оптимізована модель LSTM v3 демонструє стійку точність MAPE у межах **1.5% – 3.1%**, коефіцієнт детермінації R² = 0.92, що відповідає промисловим стандартам точності для систем оперативного планування навантаження.

6. **Впроваджено DevOps-практики.** Налаштовано повний CI/CD-конвеєр (GitHub Actions → Docker → Render.com). Пройдено аудит безпеки (Bandit, detect-secrets). 74 автоматизовані тести пройдено без помилок.

7. **Зібрано матеріал для кваліфікаційної роботи.** Матеріали звіту з практики повністю відповідають структурі майбутньої кваліфікаційної роботи та будуть використані при написанні її основних розділів (Розділ 1 — аналітика, Розділ 2 — технічне завдання, Розділ 3 — реалізація).

**Пропозиції щодо вдосконалення:**

- Розширення прогнозної моделі за рахунок включення даних про генерацію відновлюваних джерел енергії (ВДЕ) у вектор ознак;
- Інтеграція автоматичного підбору гіперпараметрів (Bayesian Optimization) для адаптації моделі до нових підстанцій без повного перенавчання;
- Розробка RESTful API для інтеграції платформи EnergyMonitor-OLAP з існуючими корпоративними SCADA-системами операторів систем розподілу.

Вважаю, що завдання переддипломної практики виконані у повному обсязі. Отримані теоретичні та практичні результати є достатньою основою для написання та успішного захисту кваліфікаційної роботи бакалавра.
