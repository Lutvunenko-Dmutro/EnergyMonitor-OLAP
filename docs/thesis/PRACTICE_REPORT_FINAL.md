# РОЗДІЛ 2. ОБҐРУНТУВАННЯ АКТУАЛЬНОСТІ ТЕМИ ПРАКТИКИ

## 2.1. Актуальність теми дослідження

Сучасна парадигма розвитку мегаполісів у межах концепції Розумного міста (Smart City) вимагає докорінного перегляду підходів до управління сталістю та ефективністю енергетичної інфраструктури. В умовах стрімкої урбанізації та зростання пікових навантажень на енергосистеми, традиційні методи пасивного моніторингу демонструють свою неспроможність забезпечити необхідний рівень надійності. 

Критичною необхідністю стає перехід до проактивного управління енергосистемами на основі інтелектуального аналізу даних та предиктивного обслуговування (Predictive Maintenance). Центральне місце в даній трансформації займає концепція Цифрових двійників (Digital Twin) та застосування рекурентних нейронних мереж архітектури LSTM, що відкриває нові можливості для високоточного прогнозування енергоспоживання.

## 2.2. Характеристика основних параметрів дослідження

**Об’єкт дослідження:** Процеси моніторингу та предиктивного аналізу споживання електроенергії в інфраструктурі Smart City.

**Предмет дослідження:** Методи глибокого навчання (LSTM), гібридне прогнозування, архітектура OLAP та програмні засоби побудови цифрових двійників енергомереж.

**Мета роботи:** Створення інтелектуальної SaaS-платформи для високоточного прогнозування, симуляції фізичних процесів та візуалізації енергетичних потоків у реальному часі.

## 2.3. Методи дослідження

Для досягнення поставленої мети під час проходження практики було обрано та використано такі методи:
1.  **Аналітичний метод:** вивчення науково-технічної літератури та існуючих програмних рішень у сфері Smart Grid.
2.  **Математичне моделювання:** застосування рекурентних нейронних мереж (LSTM) для аналізу часових рядів.
3.  **Метод системного аналізу:** проєктування багатошарової архітектури інформаційної системи.
4.  **Експериментальний метод:** проведення тестування розроблених моделей на еталонних наборах даних (Dataset PJM Interconnection).

# РОЗДІЛ 3. ОГЛЯД ІСНУЮЧИХ РІШЕНЬ ТА ОБҐРУНТУВАННЯ ВИБОРУ ТЕХНОЛОГІЙ

## 3.1. Концепція Smart City та інтелектуальних енергосистем

У ході дослідження встановлено, що концепція Розумного міста (Smart City) базується на глибокій інтеграції інформаційно-комунікаційних технологій (ІКТ) у всі сфери життя громади. Центральним елементом є **Smart Grid (інтелектуальні мережі)** — системи, що забезпечують двосторонній обмін як електроенергією, так і даними між постачальником та споживачем.

Основними проблемами сучасних мереж є нерівномірність навантаження та виникнення пікових періодів споживання. Для їх вирішення було проаналізовано такі підходи:
*   **Реактивний підхід:** реакція на дисбаланс потужності вже після його виникнення (має низьку ефективність).
*   **Проактивне управління:** прийняття рішень на основі предиктивних моделей майбутнього стану мережі.

## 3.2. Аналіз методів прогнозування часових рядів

Під час практики проведено порівняльний аналіз математичного апарату для прогнозування енергоспоживання:

1.  **Статистичні методи (ARIMA, SARIMA):** Демонструють високу точність на стабільних даних, але нездатні враховувати складні нелінійні залежності та зовнішні фактори (погоду).
2.  **Рекурентні нейронні мережі (RNN):** Дозволяють моделювати послідовності, проте страждають від проблеми затухання градієнта при довгих часових інтервалах.
3.  **Архітектура LSTM (Long Short-Term Memory):** Обрана як основна для проєкту, оскільки завдяки механізму гейтів (Gates) здатна вибірково зберігати важливу інформацію на довгих часових інтервалах та ігнорувати шуми.

## 3.3. Вибір технологічного стеку проекту

На основі аналізу потреб системи EnergyMonitor-OLAP обґрунтовано вибір таких засобів реалізації:
*   **Мова програмування Python:** за рахунок потужних бібліотек для AI (TensorFlow, Keras).
*   **СУБД PostgreSQL (Neon Cloud):** для забезпечення OLAP-аналітики та суворої типізації телеметрії.
*   **Концепція Digital Twin:** для створення віртуальної копії фізичних об’єктів (трансформаторів) та розрахунку їх деградації.
*   **Streamlit:** для швидкої побудови інтерактивних дашбордів підтримки прийняття рішень.

# РОЗДІЛ 4. ОПИС ПРОЄКТНИХ РІШЕНЬ ТА ПРОГРАМНОЇ РЕАЛІЗАЦІЇ

## 4.1. Архітектура інтелектуальної системи EnergyMonitor-OLAP

У ході виконання індивідуального завдання було спроєктовано 4-рівневу архітектуру системи, що базується на принципах модульності та масштабованості:

1.  **Рівень даних (OLAP Layer):** Реалізований на базі хмарної PostgreSQL (Neon). Використовує схему «зірка» (Star Schema) для швидкої агрегації телеметрії підстанцій.
2.  **Інтелектуальне ядро (AI Layer):** Складається з моделей LSTM v3, які виконують прогноз навантаження на основі 24-годинного «вікна» історичних даних.
3.  **Фізичний рушій (Digital Twin):** Модуль `physics.py`, що розраховує теплову деградацію обладнання та втрати в лініях за законом Джоуля-Ленца.
4.  **Користувацький інтерфейс (UI Layer):** Веб-дашборд на базі Streamlit для візуалізації KPI, прогнозів та геопросторових даних.

## 4.2. Опис бази даних та інформаційного забезпечення

Інформаційне забезпечення системи базується на реляційній моделі даних. Основні сутності включають:
*   **Substations:** Реєстр енергетичних об'єктів та їх характеристик.
*   **LoadMeasurements:** Таблиця фактів, що містить мільйони записів телеметрії.
*   **WeatherReports:** Дані про температуру та погодні умови для врахування кореляції навантаження.

## 4.3. Реалізація предиктивної моделі LSTM

Для реалізації прогнозування було розроблено нейронну мережу з такою структурою:
*   **LSTM шар (128 юнітів):** виокремлення часових ознак.
*   **Dense шари з ReLU активацією:** нелінійна трансформація.
*   **Тригонометричне кодування часу:** використання sin/cos функцій для годин та днів тижня, що забезпечує безперервність циклічних процесів.

За результатами тестування на еталонному наборі даних PJM Interconnection досягнуто похибку прогнозу **MAPE в діапазоні 1.5% – 3.1%**, що підтверджує ефективність обраних рішень.
