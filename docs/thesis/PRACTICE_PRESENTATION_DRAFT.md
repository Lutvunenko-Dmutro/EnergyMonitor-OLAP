### 🔲 Слайд 3. Теоретичний аналіз та обґрунтування ШІ-моделі
* **Вміст слайду (Візуальна частина):**
  * **Порівняльна таблиця методів:**
    * *ARIMA/SARIMA:* класичні статистичні, лінійні, слабкі на великих горизонтах.
    * *XGBoost/Random Forest:* чутливі до часових лагів, вимагають ручного Feature Engineering.
    * *LSTM (Long Short-Term Memory):* стабільні на нелінійних даних, стійкі до шумів телеметрії.
  * **Математичний базис моделі:**
    * Механізм гейтів: $f_t$ (Forget), $i_t$ (Input), $o_t$ (Output), стан комірки $C_t$.
    * Розрахунок похибки через робастну функцію втрат **Huber Loss** для мінімізації впливу аномальних викидів телеметрії.

* **🗣️ Текст виступу (Що говорити):**
  > «На етапі теоретичного аналізу було проведено порівняння класичних статистичних моделей та сучасних методів глибокого навчання. Встановлено, що традиційні статистичні підходи, такі як моделі ARIMA або сезонні моделі SARIMA, не здатні повноцінно враховувати складні нелінійні залежності та довгострокові шаблони споживання без складної попередньої обробки даних. Тому було науково обґрунтовано вибір нейронної мережі архітектури Long Short-Term Memory, яка ефективно виявляє нелінійні патерни та зберігає довгострокову інформацію за допомогою впровадженої системи фільтрів.
  > 
  > Особливістю математичного забезпечення розробленої моделі є три ключові рішення. По-перше, для навчання моделі використано функцію втрат Huber Loss, яка поєднує переваги класичних похибок, що забезпечує стабільність навчання при наявності шумів у телеметрії.
  > 
  > По-друге, застосовано гармонійне кодування часових ознак через синуси та косинуси для представлення добових та тижневих циклів, що допомагає моделі легко розуміти циклічність часу.
  > 
  > По-третє, для захисту від перенавчання моделі застосовано перевірені методи: інструмент Дропаут, який тимчасово вимикає частину нейронів, змушуючи мережу будувати стійкі зв’язки; регуляризацію ель-два, яка утримує коефіцієнти моделі маленькими для плавної реакції на шуми телеметрії; та алгоритм ранньої зупинки навчання для завершення тренування на піку точності.»

### 🔲 Слайд 4. Архітектура та інформаційне забезпечення EnergyMonitor-OLAP
* **Вміст слайду (Візуальна частина):**
  * **Блок-схема 4-рівневої архітектури:**
    1. *Presentation Layer (UI):* Веб-інтерфейс на базі Python Streamlit.
    2. *Business Logic & AI Layer:* Предиктивний модуль LSTM + Фізичний рушій симуляції.
    3. *Data Layer (OLAP):* Хмарна СУБД PostgreSQL Neon. Схема даних «Зірка» (Fact/Dimension tables).
    4. *DevOps & Infrastructure:* CI/CD-конвеєр (GitHub Actions) + Docker.
  * Схема реляційних зв'язків бази даних та швидкість OLAP-запитів.
