# 🧠 Ядро системи (`src/`)

Ця папка містить весь основний вихідний код платформи EnergyMonitor-OLAP. Архітектура побудована за модульним принципом (Clean Architecture), що дозволяє легко масштабувати систему та підтримувати її стабільність.

## 🏗️ Структура директорії

### 1. `app/` (Application Layer)
Містить глобальні конфігурації додатку, управління життєвим циклом та спільні типи даних (Types/Enums).

### 2. `core/` (Core Analytics & Engines)
Математичне та логічне серце системи.
- **`analytics/`**: Високопродуктивні алгоритми агрегації даних, фільтрації та кластеризації (K-Means).
- **`database/`**: Data Access Layer (DAL) для роботи з PostgreSQL через SQLAlchemy.
- **`diagnostics/`**: Система аудиту безпеки та здоров'я коду в реальному часі.
- **`physics.py`**: Фізичний рушій (Digital Twin), що розраховує втрати потужності, зміну опору ліній та вплив погоди.

### 3. `ml/` (Machine Learning Pipeline)
Підсистема для прогнозування часових рядів та предиктивної аналітики.
- Реалізує як статистичні моделі (ARIMA), так і глибокі нейронні мережі (LSTM).
- Містить автоматизовані пайплайни тренування, розрахунку метрик та серіалізації моделей (`ONNX`/`H5`).
- Включає папку `models/`, де зберігаються готові до продакшену ваги та скейлери.

### 4. `services/` (Business Logic & Simulation)
Фонові сервіси та виконання бізнес-логіки.
- **Data Mining**: Аналіз трендів та пошук аномалій.
- **Data Generator**: Симулятор телеметрії, який імітує поведінку реальної енергомережі для стрес-тестів.
- **Migrations**: Сервіси міграції схем та заповнення бази даних.

### 5. `ui/` (User Interface - Streamlit)
Фронтенд шар на базі Streamlit, організований для максимального перевикористання.
- **`components/`**: Перевикористовувані UI-віджети (картки, інтерактивні графіки Plotly).
- **`segments/`**: Більші структурні блоки (Sidebars, Dashboards, Live KPI grids).
- **`views/`**: Повноцінні екрани додатку (Map View, Finance, Generation, Advanced Analytics).

### 6. `utils/` (System Utilities)
Допоміжні інструменти.
- Менеджери кешування (`cache_manager.py`).
- Універсальні обробники помилок та валідатори системи.
- Єдина конфігурація логування (`logging_config.py`).

## 🚀 Технологічний стек
- **Мова**: Python 3.10+ (зі суворою типізацією)
- **Machine Learning**: TensorFlow / Keras, Scikit-Learn, Statsmodels
- **Обробка даних**: Pandas, NumPy
- **Візуалізація**: Plotly, Streamlit
- **База даних**: PostgreSQL, SQLAlchemy, Psycopg2
